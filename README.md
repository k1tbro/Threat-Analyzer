# Анализатор угроз — Итоговое домашнее задание

## Краткое описание работы скрипта

Скрипт `threat_analyzer.py` — это инструмент для сбора, анализа данных об угрозах и формирования отчётов. Он выполняет четыре основных этапа:

### Этап 1. Сбор данных
Скрипт получает данные из **двух и более источников**:
- **Логи Suricata** (`logs/suricata/eve.json`) — события сетевой безопасности (алерты, DNS-запросы)
- **Логи попыток входа** (`logs/auth/auth_attempts.json`) — неудачные попытки аутентификации
- **CVE API** — данные об уязвимостях (CIRCL API или локальный файл `data/cve_sample.json`)

### Этап 2. Анализ данных
- Анализирует логи: находит подозрительные IP-адреса, частые DNS-запросы к вредоносным доменам
- Анализирует CVE: выявляет уязвимости с высоким CVSS-баллом (≥ 7.0)
- Объединяет данные из разных источников для выявления угроз

### Этап 3. Реагирование на угрозы
При обнаружении угроз скрипт:
- Выводит сообщения о найденных угрозах в консоль
- Имитирует блокировку подозрительных IP (вывод команды `iptables`)
- Рекомендует добавление доменов в blacklist

### Этап 4. Формирование отчёта и визуализация
- Сохраняет отчёт в формате **JSON** (`report.json`) и **CSV** (`report.csv`)
- Строит график с топ-5 подозрительных IP и распределением CVSS-баллов
- Сохраняет график в файл **threat_analysis.png**

## Запуск

```bash
# Установка зависимостей
pip install -r requirements.txt

# Запуск скрипта
python threat_analyzer.py
```
